# CAZ Parser - Ежедневный парсер новостей кафедр

Автоматическое ежедневное собирание актуальных новостей с официальных источников 40 кафедр физического факультета МГУ (сайты, VK, Telegram) и отправка сводки в единый Telegram чат для медийщиков проекта КАЯ профкома физфака:https://vk.com/ffkaya.

---

## 🤖 1. Разработка

### 1.1 Настройка окружения

Рекомендуется создать виртуальное окружение для проекта и установить в него зависимости, в том числе
для разработки:

1. Перейдите в папку проекта

2. Создайте виртуальное окружение командой и активируйте его:
    ```console
    foo@bar:~$ python3 -m venv venv
    foo@bar:~$ source ./venv/bin/activate  # На MacOS и Linux
    foo@bar:~$ venv\Scripts\activate  # На Windows
    ```

3. Установите библиотеки
    ```console
    foo@bar:~$ pip install -Ur requirements.txt -r requirements.dev.txt
    ```

### 1.2 ENV file description

- DB_DSN = "postgresql+psycopg2://user:pass@localhost:5432/dbname" - строка подключения к БД
- WRITER_TOKEN = "..." - токен бота для отправки сообщений
- MY_CHAT_ID = "..." - id чата,  который необходимо отпралять сообщение
- TG_API_ID = ... - данные бота парсера
- TG_API_HASH = '...' - данные бота парсера
- PHONE_NUMBER = '...' - данные бота парсера
- VK_TOKEN = "..." - сервисный ключ доступа VK

### 1.3 Настройка базы данных
Проект использует PostgreSQL и Alembic для миграций.

1. Создайте базу данных в PostgreSQL и запишите ее название в env файл.
2. Примените миграции (создание таблиц):
   ```console
   foo@bar:~$ alembic upgrade head
3. Перенесите данные из эксель файла в БД:
   ```console
   foo@bar:~$ python -m data.seed_db

## 🔍 2. Источники информации и парсинг

### 2.1 Источники

| Источник | Количество | Способ парсинга | Примечания |
|----------|-----------|-----------------|-----------|
| **Сайты кафедр** | 40 | Отдельная парсилка на каждый сайт | Своя логика для каждого |
| **VK группы/каналы** | 20 | VK API | Использовать VK API |
| **Telegram каналы** | 8 | Telethon UserClient | Использовать UserBot |

### 2.2 Процесс парсинга

```
17:00 каждый день
    ↓
Database: читает все кафедры из базы данных
    ↓
ParserManager: создаёт задачи для всех источников
    ├─ WebsiteParser
    ├─ VkParser
    └─ TgParser
    ↓
Сбор результатов
    ↓
TextComposer: форматирует одно большое сообщение
    ↓
WriterBot: Отправляет основное сообщение в основной чат
    ↓
DatabaseUpdater: обновляет last_news_date в БД
```

## 📊 3. Структура данных и БД

### Таблица departments

**Описание полей:**
- `id` — уникальный идентификатор
- `name` — название кафедры (например, "Физика", "Математика")
- `contact` — контакт кафедры (ФИ сотрудника, телефон, email)
- `website_url` — URL сайта кафедры
- `vk_url` — URL группы/канала VK (может быть NULL)
- `tg_url` — URL канала Telegram (может быть NULL)
- `last_news_date` — дата последней найденной новости
- `updated_at` — когда последний раз обновлена запись

## 🌐 4. Типы парсеров

### 4.1 web_parser

**Класс:** `web_parser`  
**Ответственность:** Парсинг сайтов кафедр (Selenium)

```python
class WebsiteParser(BaseParser):
    async def parse(self, department: Dict) -> Optional[Dict]:
        """
        Парсит сайт кафедры и извлекает последнюю новость
        
        Returns: {
            "title": "Название кафедры",
            "source": "https://example.com",
            "contact": "Иван Петров, +7-900-123-45-67",
            "text": "первые 100 символов",
            "date": "2026-01-25",
            "link": "https://example.com/news/123"
        }
        """
```

**Особенности:**
- У каждого сайта своя логика парсинга

**Примеры реализации:**
```
parsers/
├── parsers/
│   ├── physics_parser.py     # логика для сайта физики
│   ├── math_parser.py        # логика для сайта математики
│   └── ...
└── website_parser.py         # main класс, который выбирает нужный парсер
```

### 4.2 vk_parser

**Класс:** `vk_parser`  
**Ответственность:** Парсинг VK групп/каналов через официальный VK API

```python
class VkParser(BaseParser):
    async def parse(self, department: Dict) -> Optional[Dict]:
        """
        Парсит VK группу/канал и извлекает последний пост
        
        Returns: {
            "title": "Название кафедры",
            "source": "https://vk.com/physics_group",
            "contact": "Иван Петров, +7-900-123-45-67",
            "text": "первые 100 символов поста",
            "date": "2026-01-25",
            "link": "https://vk.com/wall_123456_789"
        }
        """
```

**Особенности:**
- Использует VK API (нужен access_token)

### 4.3 tg_parser

**Класс:** `tg_parser`  
**Ответственность:** Парсинг Telegram каналов через Telethon

```python
class TelegramParser(BaseParser):
    async def parse(self, department: Dict) -> Optional[Dict]:
        """
        Парсит Telegram канал и извлекает последнее сообщение
        
        Returns: {
            "title": "Название кафедры",
            "source": "https://t.me/physics_channel",
            "contact": "Иван Петров, +7-900-123-45-67",
            "text": "первые 100 символов сообщения",

**Особенности:**
- Сессия авторизуется один раз и переиспользуется (НЕ удаляем .session)
